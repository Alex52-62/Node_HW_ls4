"Проваливание" в поддиректории достигается за счёт рекурсии (стр 46 - return  await walk(); )
Для проверки создал "матрёшку" - catalogue1/catalogue2/catalogue3 в каждой лежит простой текстовый файл

Поиск в файле достигается с помощью ввода аргументов, к примеру: node HW.js --p=This
при обращении к файлу CAT1(2,3).txt выведет [ 'This' ], те покажет наличие паттерна,
а вот если прочитать HW.js то там найдётся уже 3 this - [ 'this', 'this', 'this' ] и тд

node HW.js --p=hhh - покажет отсутствие паттерна, те - null

поиск осуществляется с помощью RegExp(pattern, flags) и используются следующие флаги:
i - игнорирование регистра при сопоставлении
g - глобальное сопоставление
m - сопоставление по нескольким строкам; символы начала и конца (^ и $) начинают работать 
    по нескольким строкам (то есть, происходит сопоставление с началом или концом каждой строки 
    (строки разделяются символами \n или \r), а не только с началом или концом всей вводимой строки)
